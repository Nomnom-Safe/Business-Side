# Diagram Dependencies

The diagrams follow a logical chain. Work should progress in this order:

**P-Diagram → Failure Modes → Root Cause Analysis → Prioritization → Resolution**

More specifically:

- **P-Diagram drives Failure Modes**
- **Failure Modes drive Fishbone and 5 Whys**
- **Failure Modes also drive the Problem Assessment Matrix**
- **Problem Matrix drives the Problem Resolution Plan**

Because of these dependencies, the **P-Diagram and Failure Modes must be completed first**.
